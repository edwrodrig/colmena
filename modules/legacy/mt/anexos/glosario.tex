\paragraph{API : } (\textbf{A}plication \textbf{P}rogramming \textbf{I}nterface) Se refiere a un conjunto de funciones y métodos que se utilizan para facilitar la programación. Algunas aplicaciones incluyen APIs para permitir al usuario personalizar ciertos aspectos del software.
\paragraph{C++ : } Lenguaje de programación orientado a objetos. Los ejecutables generados por sus compiladores son más rápidos que los de los lenguajes interpretados por el mismo hecho de no necesitar un intérprete.
\paragraph{FIFO : } (\textbf{F}irst \textbf{I}n \textbf{F}irst \textbf{O}ut) Es un comportamiento de los elementos en una estructura de datos donde el primer elemento en entrar es el primero en salir.
\paragraph{FLOPS : } (\textbf{FL}oating Point \textbf{O}peration \textbf{P}er \textbf{S}econd) Unidad de medida de procesamiento. Una unidad corresponde a una operación de punto flotante por segundo. 

\paragraph{HTTP : } (\textbf{H}yper\textbf{T}ext \textbf{T}ransfer \textbf{P}rotocol) Es el protocolo usado para transacciones en Internet. Define la sintaxis y la semántica que deben utilizar los elementos de software de la arquitectura web.
\paragraph{IP : } (\textbf{I}Internet\textbf{P}rotocol) Es el protocolo a nivel de red usado en Internet para la comunicación de datos. También se suele referir con esta sigla a las direcciones que poseen los computadores en una red IP.
\paragraph{LAN : } (\textbf{L}ocal \textbf{A}rea \textbf{N}etwork) Es una categoría de red de computadora que cubre un area local. Generalmente cubren areas como hogares u oficinas, pero el concepto es relativo y puede cambiar según el contexto. 
\paragraph{NAT : } (\textbf{N}etwork \textbf{A}ddress \textbf{T}ranslator) Mecanismo en el que un computador con una dirección y puerto interno puede obtener una dirección y puerto externo. Este mecanismo está implementado en la mayoría de los \textbf{firewall} y \emph{routers}.
\paragraph{P2P : } (\textbf{P}eer \textbf{2} \textbf{P}eer) Forma de interacción entre elementos de una red donde no existe jerarquía. Esto se traduce generalmente en que no hay entidades coordinadores como lo son los servidores.
\paragraph{Perl : } Es un lenguaje de programación interpretado. Se caracteriza por tener componentes sintacticos que facilitan el procesamiento de textos.
\paragraph{PHP : } (\textbf{P}HP \textbf{H}ypertext \textbf{P}reprocessor) Es un lenguaje de programación interpretado que suele usarse en páginas web dinámicas.
\paragraph{Qt : } Framework de programación en C++. Se caracteriza por incluir un conjunto de clases preimplementadas, un método de paso de mensajes entre ellas y facilitar la portabilidad de una aplicación entre sistemas operativos.
\paragraph{SaaS : } (\textbf{S}oftware \textbf{a}s \textbf{a} \textbf{S}ervice) Visión en la cual se ve el software como un servicio. Este paradigma es visto generalmente en Internet, donde se ven una gran cantidad de aplicaciones online que son ofrecidas como servicios a los usuarios.
\paragraph{SQL : } (\textbf{S}imple \textbf{Q}uery \textbf{L}anguaje) Es un lenguaje declarativo usado en la interacción con bases de datos relacionales. Se usa tanto para operar sobre las tablas como también para hacer consultas sobre éstas.
\paragraph{SQLite3 : } Implementación en C de una base de datos relacional SQL. Se diferencia de otras base de datos en que almacena la totalidad de la base de datos en un sólo archivo.
\paragraph{TCP : }(\textbf{T}ransmission \textbf{C}ontrol \textbf{P}rotocol) El protocolo de control de transmisión está pensado para ser utilizado como un protocolo de transmisión de datos en forma de paquetes host a host orientado a la conexión. Esto significa que este protocolo se encarga de aspectos como el orden de llegada de los paquetes enviados, la integridad de la información recibida y la persistencia de la conexión entre los hosts. Si bien estas características sobre el control de la transmisión sirven para obtener un medio de comunicación confiable, los mecanismos para hacerlo posible significan un costo en la tasa de transferencia pues implica la transmisión de paquetes extras a la transmisión de datos en sí.
\paragraph{UDP : }(\textbf{U}ser \textbf{D}atagram \textbf{P}rotocol) El protocolo de datagrama de usuario está pensado para comunicar dos procesos y que se envíen datos con el mínimo de mecanismos anexos. Esto significa que el protocolo se encarga simplemente del despacho del mensaje hacia algún destino, despreocupándose de aspectos tales como la recepción efectiva del mensaje, el orden de llegada y la integridad de éstos. Esto convierte a UDP en un protocolo de transmisión poco confiable pero liviano.
\paragraph{URL : } (\textbf{U}niform \textbf{R}esource \textbf{L}ocator) Es un formato que se usa para nombrar diversos recursos en Internet como lo son las páginas web.
\paragraph{WAN : } (\textbf{W}ide \textbf{A}rea \textbf{N}etwork) Es una categoría de red de computadora que se diferencia de una \emph{LAN} en que cubre un area geográfica más amplia.
\paragraph{XML: } (E\textbf{X}tended \textbf{M}arkup \textbf{L}anguaje) Es un lenguaje que especifica información mediante un formato basado en tags anidados. Es muy común en muchos formatos de archivos y en la web.
\paragraph{ZLIB : } Es un biblioteca de código abierto para compresión de datos. Actualmente es incluida nativamente como una función de Qt para la compresión de arreglos de bytes.
