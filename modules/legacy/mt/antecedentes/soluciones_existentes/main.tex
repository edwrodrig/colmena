Además se contrasta el cluster virtual contra la solución ofrecida por la computación de nube.

\subsection{Computación de nube}
Como se mencionó anteriormente, cuando se habla de computación de nube no se está hablando de una tecnología en particular, sino de una solución ofrecida como un servicio. Elegir entre pagar por un servicio o implantar tecnología se puede traducir a la pregunta : \emph{¿Es preferible cultivar mi comida o comprarla en el supermercado?}. Dentro del contexto del proyecto esta pregunta sería : \emph{¿Es preferible usar el cluster virtual o simplemente pagar por el servicio de cómputo?}.
Para responder a esta pregunta hay que remitirse al objetivo principal del proyecto que es \emph{''aprovechar el poder de cómputo de los computadores ociosos''}. De esta sentencia se deduce que el cluster virtual está pensado para aprovechar un potencial no utilizado. Si existe tal potencial entonces ¿por qué se tendría que pagar adicionalmente por él?. En resumen, la computación de nube resuelve el problema de necesidad de poder de cómputo mediante recursos económicos, en cambio el sistema de este proyecto resuelve este problema mediante el uso de recursos subutizados, lo que significa una solución más óptima.
